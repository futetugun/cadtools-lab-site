AutoCAD Productivity Plugins

CAD図面作業を、 小さなプラグインで 確実に短縮。

文字修正、図理チェック、社内CADルール対応。建築設備設計の現場で繰り返し発生する作業を、軽量な AutoCAD 向けプラグインとして整理します。

AutoCAD向け AutoLISP中心 会社版ライセンス対応 日本語運用環境を想定
Products

今ある機能と、追加予定のプラグイン。

最初の実験対象は文字変換プラグインです。今後は同じ販売・ライセンス・検証フローで、別の CAD 補助プラグインを追加できます。
検証済み

Text Cleanup

図面内の全角数字、全角英字、括弧、半角カタカナを選択範囲だけ変換します。

  • 注記・設備表記の揺れを整理
  • TEXT、MTEXT、属性文字を対象
  • 会社版ライセンスに対応
追加予定

Attribute Tools

ブロック属性、機器番号、盤名称、図面内の定型文字をまとめて確認・修正するツール群。

  • ブロック属性の抽出
  • 社内ルールに沿った表記チェック
  • 設備図面向けの一括修正
追加予定

Drawing Check

図面提出前の文字、層、尺度、注記ルールを確認する軽量チェックツール。

  • 提出前チェックの省力化
  • チェックリストの自動化
  • 会社ごとの CAD 基準に対応
Example

何をするプラグインか、すぐ分かる表示。

製品ページでは、専門用語よりも「変換前」「変換後」を前面に出します。担当者が一目で用途を判断できる構成です。
Before 図面注記
ＡＢＣ１２３（テスト） 盤ＮＯ．１２（１Ｆ） ｶﾝｷﾌｧﾝ（ＥＦ－１）
After Plugin Result
ABC123(テスト) 盤NO.12(1F) カンキファン(EF-1)

この最初のプラグインでできること

  • 全角数字を半角へ変換
  • 全角英字を半角へ変換
  • 括弧の全角/半角を整理
  • 半角カタカナを全角へ統一

会社で使う時に必要な前提

  • 試用版と会社版を分けて管理
  • 問い合わせ窓口を固定
  • 導入前に対象環境を確認
  • 追加仕様は別見積に分離
Proof

小さな機能でも、会社導入では判断材料が必要です。

個人向けの安価なツールと違い、会社版では「何に効くか」「どう配るか」「誰が対応するか」を先に見せる方が成約しやすくなります。
01

用途を一行で言える

図面内の文字表記を揃える、という説明に落とし込みます。複雑な機能説明は二段目で十分です。

02

対象オブジェクトを明示する

TEXT、MTEXT、属性文字、マルチ引出線など、実際に変換できる範囲を先に見せます。

03

試用から本導入までを短くする

試用版、会社版、定制開発を分けることで、稟議前のテストと正式導入を同じ説明線上に置けます。

Workflow

導入は短く、確認は明確に。

小さなプラグインでも、会社で使う場合は配布、ライセンス、検証ログが必要です。導入手順を固定し、顧客側の不安を減らします。
01

読み込み

AutoCAD にプラグインを読み込みます。

02

ライセンス設定

発行された会社版コードを設定します。

03

状態確認

製品名、有効期限、端末情報、キーを確認します。

04

実行

対象を選択し、必要な処理だけ実行します。

Demo

動画は60秒以内で、機能だけを見せる。

ページには説明を用意しても、実際の AutoCAD 操作を動画で見せた方が早いです。`assets/plugin-demo.mp4` として置き換えれば、そのまま再生できます。

推奨構成は「読み込み → ライセンス確認 → 対象選択 → 実行 → 変換結果」です。説明より画面を見せる方が早いです。
Custom Development

会社ごとの CAD ルールに合わせて拡張。

単品プラグイン販売だけでなく、社内ルールに合わせた定制開発を提案できます。小さな機能から始め、業務改善の入口にします。
